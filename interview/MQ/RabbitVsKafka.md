# Kafka

+ 日志分析，处理
+ 实时流数据处理

# RabbitMQ

适用于传统消息中间件的场景，如任务投递。

## 区别

### Replay

RabbitMQ: 消息消费完成后被丢弃
Kafka: 消息可重复消费

### Routing

RabbitMQ: 支持灵活的路由配置
Kafka: 不支持
